MySQL vs MongoDB comparison (comparing traditional SQL with NoSQL DBMS)

Abstract

I compared MongoDB which is a NoSQL or non-relational DBMS with MySQL which is a SQL or relational DBMS. This work is important as both relational and non-relational databases are used worldwide for many critical tasks ranging from business to healthcare. For the fairest comparison possible, I used the Yahoo Cloud Serving Benchmark and ran it a few times on each database for results. I compared both latency and throughput. Latency is essentially how long it takes to complete an operation and throughput is the number of operations completed per second. With 1,000 records, I found that MySQL was outperforming MongoDB in both latency and throughput while with 100 million records, MongoDB showed much better throughput and outperformed MySQL with latency. After looking over the findings, it appears that MongoDB has improved performance as the number of records increases. Depending on the use-case, MySQL seems to be best for data consistency while MongoDB appears to be better optimized for latency.
